# ETIC 手动测试条目 · M2（起卦 + 静态排盘）/ M3（占卜动画）

> 适用范围：在 **macOS + Xcode** 上对 iOS App（`App/`）做手动验收。
> 排盘引擎 `DivinationEngine` 的自动化测试见 `Packages/DivinationEngine`（`swift test`），**不在本清单内**——本清单只覆盖 SwiftUI 客户端的界面、交互与动画。
> 阶段⑥（LLM 流式解读）属 M4，不在本清单内（M2/M3 中「请大师解读」按钮应为禁用占位）。

---

## 0. 环境与前置条件

| 项 | 要求 |
| --- | --- |
| 系统 | macOS（Apple Silicon 或 Intel 均可） |
| Xcode | 15 及以上 |
| 模拟器 | iOS 16.0 及以上（覆盖最低部署版本 16.0） |
| 真机（可选） | iPhone（iOS 16+）——**摇一摇与 CoreHaptics 触觉只能在真机验证**，模拟器不支持 |
| 工具 | [XcodeGen](https://github.com/yonyz/XcodeGen)（`.xcodeproj` 不入库，需生成） |

### 0.1 生成工程并运行

```bash
brew install xcodegen          # 若未安装
cd App
xcodegen generate              # 由 project.yml 生成 ETIC.xcodeproj
open ETIC.xcodeproj            # 选 iOS 16+ 模拟器，Cmd+R 运行
```

### 0.2 验证矩阵（建议至少覆盖）

- [ ] iOS 16.x 模拟器（最低部署版本）
- [ ] 最新 iOS 模拟器（如 iOS 17/18）
- [ ] 浅色外观（本 App 仅做了浅色水墨主题，深色下也应可读、无不可见文字）
- [ ] 一台真机（仅用于 M3 的摇一摇 + 触觉，见 §3.4 / §3.5）

### 0.3 用例约定

- 编号：`TC-<模块>-<序号>`。
- 每条含「步骤 / 预期结果」，前面的 `[ ]` 勾选框用于记录 **通过 / 不通过**。
- 「起卦方法」分段选项（即枚举 `rawValue`）：**铜钱（默认）/ 报数 / 时间 / 随机**。M2 阶段铜钱法用随机背数模拟（真正摇一摇在 M3）。

## 3. M3 · 占卜动画（RitualView）

> 入口：在起卦页点「起卦」后进入。动画为「演出」，**结果已由引擎算定**，动画不改变盘面。

### 3.1 阶段编排（罗盘 → 摇卦 → 成卦 →〔动爻变卦〕→ 盘面）

- [ ] **TC-ANIM-01 起卦准备（罗盘）**
  预期：进入后显示旋转的「太极 + 八卦」罗盘（Canvas 绘制），文案「静心默念所问之事」，若填了问题则显示问题；底部「开始摇卦」按钮。罗盘缓慢平滑旋转，无明显掉帧。

- [ ] **TC-ANIM-02 进入摇卦**
  步骤：点「开始摇卦」。
  预期：过渡到摇卦阶段，出现三枚铜钱与「摇卦（0/6）」按钮、提示文案。

- [ ] **TC-ANIM-03 单次摇卦（点按）**
  步骤：点「摇卦」按钮。
  预期：三枚铜钱先翻滚抖动（约 0.7s），随后落定；计数 `revealed` +1（按钮显示如「摇卦（1/6）」）；对应一爻自下而上「写出」（阳爻整笔、阴爻断笔）。摇卦进行中按钮禁用、不可连点。

- [ ] **TC-ANIM-04 六爻逐爻成卦**
  步骤：连续摇卦 6 次。
  预期：六爻自下而上依次写出；满 6 次后按钮变「成卦中…」并禁用。

- [ ] **TC-ANIM-05 动爻变卦阶段（有动爻时）**
  前置：起卦结果包含动爻（老阳/老阴；可多次起卦直到出现，或用时间/数字法构造）。
  预期：满 6 爻后进入「动爻变卦」阶段，标题「动爻变卦」（朱砂色）；动爻处 `○`(老阳)/`×`(老阴) 标记闪烁数次后稳定，随后过渡到排盘页。

- [ ] **TC-ANIM-06 无动爻直达盘面**
  前置：起卦结果**无**动爻（六爻皆少阴少阳）。
  预期：满 6 爻后**不**进入变卦闪烁，短暂停顿后直接淡入排盘页。

- [ ] **TC-ANIM-07 铜钱面与爻象一致性（关键）**
  步骤：任选一次起卦，记录每爻落定时三枚铜钱的「背」数（朱砂底、标「背」字为背面；卡片底、标「字」字为字面），与最终盘面该爻的老少阴阳对照。
  预期：满足约定 **3 背=老阳、2 背=少阴、1 背=少阳、0 背=老阴**；即演出的铜钱结果与排盘页爻象**完全吻合**（与引擎 `Caster.fromCoinBacks` 一致）。

### 3.2 跳过动画

- [ ] **TC-ANIM-08 任意阶段可跳过**
  步骤：分别在「罗盘」「摇卦中途」「变卦闪烁」阶段点右上角「跳过动画」。
  预期：立即淡出动画、直达排盘页；盘面完整且与不跳过时一致（同一次起卦输入）。

- [ ] **TC-ANIM-09 跳过后盘面正确**
  预期：跳过得到的六爻、动爻、变卦与正常演出完毕的结果相同（结果由引擎决定，跳过只是省略演出）。

### 3.3 性能 / 视觉

- [ ] **TC-ANIM-10 流畅度**
  预期：罗盘旋转、铜钱翻滚、爻笔写出动画流畅（目标 ~60fps，低端机不卡死）；首屏不被动画长时间阻塞。

- [ ] **TC-ANIM-11 横竖屏 / 不同机型**
  步骤：在 SE（小屏）与 Pro Max（大屏）模拟器各跑一次。
  预期：罗盘、铜钱、爻线布局不溢出、不重叠；按钮可点。
  （注：工程限定竖屏，验证竖屏即可。）

### 3.4 摇一摇（CoreMotion，**仅真机**）

- [ ] **TC-ANIM-12 摇一摇触发摇卦**
  前置：真机；设置中「摇一摇触发摇卦」开启（默认开）。
  步骤：进入摇卦阶段，**摇晃手机**。
  预期：摇晃触发一次摇卦（等效点按按钮）；提示文案包含「摇一摇手机，或点下方按钮摇卦」。

- [ ] **TC-ANIM-13 去抖冷却**
  步骤：连续快速摇晃。
  预期：约 0.6s 冷却内不重复触发；不会一次摇晃摇出多爻。

- [ ] **TC-ANIM-14 关闭摇一摇**
  步骤：设置中关闭「摇一摇触发摇卦」→ 起卦 → 摇晃手机。
  预期：摇晃**不**触发；提示文案变为「点下方按钮摇卦」；点按仍可摇卦。

### 3.5 触觉反馈（CoreHaptics，**仅真机**）

- [ ] **TC-ANIM-15 铜钱落地触觉**
  前置：真机；设置「触觉反馈」开启（默认开）；系统未静音/未关触觉。
  步骤：摇卦使铜钱落定。
  预期：落定时有清脆的双击式触觉（铜钱叮当）。

- [ ] **TC-ANIM-16 变卦轻击触觉**
  预期：动爻闪烁时伴随轻柔脉冲触觉。

- [ ] **TC-ANIM-17 关闭触觉**
  步骤：设置关闭「触觉反馈」→ 起卦摇卦。
  预期：无触觉反馈；其余动画正常。

- [ ] **TC-ANIM-18 不支持触觉的设备降级**
  前置：较老不支持 CoreHaptics 的真机（或模拟器）。
  预期：不崩溃；退化为系统 `UIImpactFeedback`（或无），动画流程不受影响。

### 3.6 设置（RitualSettingsView）与持久化

- [ ] **TC-SET-01 三个开关**
  预期：设置页含「跳过占卜动画」「摇一摇触发摇卦」「触觉反馈」三个开关，及说明文字「动画仅为演出，不影响起卦结果」。

- [ ] **TC-SET-02 跳过动画开关**
  步骤：开启「跳过占卜动画」→ 起卦。
  预期：起卦后**直接到排盘页**，不播放罗盘/摇卦动画。

- [ ] **TC-SET-03 持久化**
  步骤：修改任一开关 → 杀掉 App → 重新启动 → 打开设置。
  预期：开关状态被记住（`@AppStorage` 持久化）。

- [ ] **TC-SET-04 系统「减弱动态」联动**
  前置：iOS 设置 → 辅助功能 → 动态效果 → 打开「减弱动态」。
  步骤：起卦。
  预期：动画降级——直接淡入到排盘页（不旋转罗盘、不逐爻演出）；设置页在系统开启时显示「系统已开启『减弱动态』，动画将自动降级」提示。

  预期：「解读」卡片说明 M4 接入；「请大师解读」按钮**禁用**不可点。

- [ ] **TC-BOARD-13 返回再起卦无残留**

## 5. 回归 / 健壮性

- [ ] **TC-REG-01 引擎契约只读**
  说明：客户端**不得**改动盘面数据。抽查：同一起卦输入（同方法、同时间、铜钱法固定背数）经 App 得到的盘面，与引擎直接计算一致。

- [ ] **TC-REG-02 长问题 / 特殊字符**
  步骤：问题填超长文本、emoji、纯空格。
  预期：不崩溃；排盘页回显合理（空格视为无问题可接受）。

- [ ] **TC-REG-03 连续多次起卦内存**
  步骤：连续起卦 ~20 次并返回。
  预期：无明显内存增长/泄漏（Xcode Memory Graph 抽查）、无卡顿累积。

- [ ] **TC-REG-04 后台/前台切换**
  步骤：动画进行中切到后台再回前台。
  预期：恢复后状态合理（动画继续或停在某阶段），不崩溃；CoreMotion/Haptics 资源在 `onDisappear` 已释放（离开页面不再监听加速度）。

- [ ] **TC-REG-05 旋转/字体放大（可选）**
  步骤：调大系统字体（动态字体）。
  预期：文字不严重截断、布局可用。
